What it is

FocusED is an Android accessibility service that watches for Instagram Reels and YouTube Shorts and fires a back-press the instant they appear — before you've had a chance to start scrolling. It doesn't block the apps, doesn't touch DMs, doesn't require an account, and has no internet permission in the manifest. One toggle, zero footprint.

UI event fires → Match pattern → Reels / Shorts? → Back press → Done

Android's Accessibility Service API lets apps observe UI events system-wide — the same mechanism screen readers use. FocusED registers as a service and listens for TYPE_WINDOW_STATE_CHANGED events. When an activity matching the Reels or Shorts fingerprint appears, it fires a back press immediately. No overlay, no network call, no storage write.

How it was built

Native Android in Java. The core is a single service class extending AccessibilityService with about 80 lines of actual logic. No third-party libraries. The APK is under 200KB — there's nothing in it except the service, a minimal settings activity, and the manifest.

The hardest part was identifying stable UI signatures for Reels and Shorts across app updates. Both Instagram and YouTube change their activity and fragment names regularly, so matching uses a combination of package name, class name patterns, and content descriptions that stay stable across versions.

What I learned

The Accessibility Service API is genuinely powerful — and genuinely alarming in the wrong hands. The permission dialog Android shows is appropriately scary: "this app can observe everything you do." Open source was the only honest answer. No internet permission was a deliberate trust signal, not an afterthought — users who care about privacy notice it immediately.
