What Building This Taught Me

Real skills from real problems

Architecture & Design

  • How frontend and backend actually talk to each other
  • Why WebSockets beat regular HTTP for real-time chat
  • Designing API endpoints that make logical sense
  • Data modeling and proper separation of concerns

Security Isn't Optional

  • Password hashing with bcrypt (never store plain text)
  • JWT authentication flows and token management
  • Keeping user data completely isolated (multi-tenancy basics)
  • Why .env files should never touch GitHub (learned this one the hard way)

Real-Time Communication

  • Streaming responses chunk-by-chunk instead of waiting
  • Managing persistent WebSocket connections
  • Async programming concepts (async/await magic)
  • Bidirectional data flow between client and server

Deployment & DevOps

  • Docker containerization ("works on my machine" → "works everywhere")
  • Environment configuration for development vs production
  • Debugging issues you can't replicate locally
  • Basic networking concepts (ports, protocols, DNS)

The Meta-Lesson

You don't need to write every line to understand the system. Architecting, testing, deploying, and iterating taught me more about real development than memorizing syntax ever could.

The value isn't in who typed the code—it's in understanding what problems you're solving and how the pieces fit together.
