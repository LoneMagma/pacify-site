BurnLab

Latest
Stable Release V1

The Sovereign Development Environment

A complete air-gapped & AI-powered development laboratory that runs entirely from a USB drive. BurnLab transforms any PC into a self contained coding environment with local LLMs, offline documentation, and professional tooling, no internet, no installation, no data left behind.

Local Intelligence

Integrated Ollama runtime with Qwen2.5-Coder for offline code generation and debugging

Universal Reference

15GB+ of offline knowledge: Wikipedia, Stack Overflow, MDN, Arch Wiki

Professional Tooling

VS Code, Python 3.11+ Scientific Stack, Git, and complete build essentials

Zero-Trace

Runs entirely from RAM and USB. No data touches the host computer

Key Features

  • One-Command Bootstrap: Entire environment deployed via single bash script!
  • Adaptive Storage: Smart compression allows full functionality on 16GB drives
  • Intuitive Aliases: ask "How do I reverse a list?" queries local LLM with offline docs
  • Layered Architecture: BunsenLabs base --> Btrfs persistence --> Kiwix knowledge --> Quantized LLMs
BunsenLabs Linux Ollama Qwen2.5-Coder Kiwix VS Code Btrfs Python

Built this because I wanted a development environment that works anywhere! School's Computer Lab, libraries or random computers that too without leaving traces or depending on internet. The entire system fits on a USB drive smaller than your thumb.

What Building This Taught Me

Real skills from real problems

Architecture & Design

  • How frontend and backend actually talk to each other
  • Why WebSockets beat regular HTTP for real-time chat
  • Designing API endpoints that make logical sense
  • Data modeling and proper separation of concerns

Security Isn't Optional

  • Password hashing with bcrypt (never store plain text)
  • JWT authentication flows and token management
  • Keeping user data completely isolated (multi-tenancy basics)
  • Why .env files should never touch GitHub (learned this one the hard way T_T )

Deployment & DevOps

  • Docker containerization ("works on my machine" implies "works everywhere")
  • Environment configuration for development vs production
  • Debugging issues you can't replicate locally
  • Basic networking concepts (ports, protocols, DNS)

Systems & Infrastructure

  • Linux system architecture and persistent storage strategies
  • File system optimization (Btrfs compression saved 40% space)
  • Offline-first design and air-gapped development environments
  • Resource management on constrained hardware (16GB USB drives)

The Meta-Lesson

You don't need to write every line to understand the system. Architecting, testing, deploying, breaking and iterating taught me more about real development than memorizing syntax ever could.

The value isn't in who typed the code! it's in understanding what problems you're solving and how the pieces fit together.
