BurnLab

Stable v1 Portable

A complete AI dev environment on a USB drive. No install, no internet, no traces.

What it is

BurnLab is a portable, self-contained development environment that runs from a USB drive. You plug it in on any compatible machine, run a single script, and you have a local AI assistant, code editor, Python runtime, and shell environment — all without installing anything on the host machine. Unplug and the host is exactly as you found it.

It's aimed at the situation where you're working on a machine you don't own or trust, or where you simply want zero footprint. Security researchers, students in labs, developers on travel laptops, people who just don't want their dev environment scattered across three machines.

How it works

The USB contains a minimal Linux environment (via a lightweight portable runtime), a bundled Ollama instance with a quantized model pre-loaded, VS Code Portable, and a set of shell scripts that wire everything together at boot. The LLM runs entirely in RAM from the drive — no model weights are ever written to the host disk. 

  ┌─── USB Drive ──────────────────────────────────────────┐
  │                                                        │
  │  runtime/          portable linux env (no install)    │
  │  ollama/           LLM engine + quantized model        │
  │  vscode-portable/  editor, settings, extensions       │
  │  workspace/        your code lives here                │
  │  burn.sh           the one script you run              │
  │                                                        │
  └────────────────────────────────────────────────────────┘
           │
           ▼  plug in + run burn.sh
  ┌─── Host Machine ────────────────────────────────────────┐
  │                                                        │
  │  model loads into RAM           nothing written to disk  │
  │  VS Code opens from USB         host FS never touched  │
  │  shell + AI ready in ~30s       unplug = clean slate   │
  │                                                        │
  └────────────────────────────────────────────────────────┘

On startup, burn.sh detects the host OS, starts the Ollama server pointed at the USB model directory, launches VS Code Portable with a pre-configured workspace, and drops you into a shell with the Python environment activated. All paths are relative to the drive mount point so nothing breaks when the drive letter or mount path changes between machines.

How it was built

The hardest engineering problem was making the startup script fully portable across Windows (PowerShell), macOS, and Linux without requiring any pre-installed dependencies. On Windows that means using PowerShell's Start-Process to bootstrap the runtime; on Unix it's a standard sh shebang. A single burn.cmd wrapper handles the Windows case so users can double-click on either OS.

Model selection was a deliberate trade-off. A 4-bit quantized 7B model fits on a 32GB drive with room for a full workspace, starts in under 30 seconds on modest hardware, and is good enough for the target use case (coding assistance, not creative writing). The setup script also supports swapping models — it's just a folder.

What I learned

Portable software is harder than it looks because every assumption you've ever made about paths, permissions, and environment variables turns out to be wrong on someone else's machine. I spent a disproportionate amount of time on Windows path handling alone — spaces in drive labels, UAC quirks, antivirus flagging the startup script as suspicious.

The project also forced me to think seriously about what "no traces" actually means in practice. Swap files, prefetch caches, Windows event logs — there are a surprising number of places the OS records that you ran a process. The current version makes a best-effort attempt at cleanup on exit, but I document the known gaps honestly in the README rather than pretending it's perfect.
